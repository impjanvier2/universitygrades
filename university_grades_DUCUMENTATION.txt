1. INTRODUCTION
The University Grades Management System is a web-based application designed to manage academic records in higher learning institutions. It replaces manual methods such as spreadsheets and paper files with a centralized digital system.

The system is built on a relational database with five main entities:
- Users: stores login credentials and access roles.
- Students: stores student personal and academic information.
- Instructors: stores staff details and departments.
- Courses: stores course information.
- Enrollments: links students, courses, instructors, and grades.

The system uses role-based access control so that:
- Administrators manage users and system data.
- Instructors manage and update grades.
- Students view their academic performance.

2. PROBLEM STATEMENT
Traditional academic record systems have several problems:
- Risk of data loss from physical files or local storage.
- Errors in manual grade calculations.
- Slow processing and release of results.
- Weak security and lack of proper access control.
- Difficulty in accessing academic records remotely.

These issues make manual systems inefficient and unreliable for modern universities.

3. OBJECTIVES

General Objective:
To develop a secure and efficient system for managing student academic records.

Specific Objectives:
- Allow instructors to enter and update grades digitally.
- Allow students to view their results online in real time.
- Automatically calculate GPA and performance results.
- Allow administrators to manage users and system data securely.

4. BENEFITS

To Students:
- Easy access to academic results anytime.
- Ability to track performance in real time.
- Improved transparency in grading.

To Instructors:
- Easier grade management.
- Reduced calculation errors.
- Less administrative workload.

To Institution:
- Improved data security.
- Reduced paperwork costs.
- More efficient academic operations.
